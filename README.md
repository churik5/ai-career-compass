# AI Career Compass

Двуязычный (EN / RU) тест-дашборд, который подбирает человеку одну из 13 AI-профессий для креаторов и фрилансеров. 12 вопросов, «угадай доход» с конфетти, персональный дашборд с навыками, ценами и ресурсами.

Сделано как подарок. Без бэкенда, без внешних API.

## Запуск

```bash
npm install
npm run dev
```

После запуска открыть: **http://localhost:5173**

## Сборка

```bash
npm run build     # production-билд в dist/
npm run preview   # локальный preview сборки
npm run lint      # ESLint
```

## Что внутри

- **Vite + React 19 + TypeScript (strict)**
- **Tailwind CSS v3** — кастомные токены (`ink`, `bone`, `amber`, `neon`, `iris`, `lime`) в `tailwind.config.js`
- **Шрифты:** Fraunces (дисплей), Geist (body), JetBrains Mono (технические акценты) — подгружаются из Google Fonts
- **framer-motion v12** — анимации, переходы между страницами
- **recharts v3** — график услуг на дашборде
- **lucide-react** — иконки профессий и ресурсов
- **canvas-confetti** — конфетти при точной догадке
- **react-router-dom v7** — роутинг
- **Своя i18n** через React Context (без `react-i18next`) — два словаря + билингвальные объекты в данных

## User flow

1. **Landing** (`/`) — хиро с фирменным знаком + список ролей + 3 шага как это работает.
2. **Quiz** (`/quiz`) — 12 вопросов, один за раз, навигация стрелками/Tab/Enter. Прогресс-бар сверху, моно-счётчик `01 / 12`.
3. **Result** (`/result`) — топ-роль с матч-процентом + 2 runner-up.
4. **Salary Guess** (`/salary`) — ключевая фишка. Пользователь вводит свою догадку о месячном доходе, получает реальную вилку, реакцию в зависимости от промаха и конфетти, если попал в диапазон или ±5%.
5. **Dashboard** (`/dashboard`) — персональная панель: описание роли, ключевые навыки, чем занимаются, услуги с ценами (график + текстовый список), ресурсы для старта, кнопки «Поделиться» / «Пройти заново».

Роуты защищены: прямой переход на `/dashboard` без пройденного теста → редирект на `/quiz`.

## Двуязычность

**Английский — источник правды, русский — перевод.** Переключатель EN / RU в правом верхнем углу (сегментный тоггл, не dropdown). Состояние хранится в памяти (не в localStorage) — при обновлении страницы сбросится на английский, как и состояние теста.

### Как редактировать тексты

| Что хотите поменять                          | Где искать                                                |
|----------------------------------------------|-----------------------------------------------------------|
| UI-строки (кнопки, заголовки, реакции и т.д.) | `src/locales/en.json` и `src/locales/ru.json`             |
| Названия, слоганы, описания профессий         | `src/data/roles.ts` → поля `name`, `slogan`, `description` |
| Навыки, что делает профессия, ресурсы         | `src/data/roles.ts` → `skills`, `whatTheyDo`, `resources`  |
| Цены на услуги и их названия                  | `src/data/roles.ts` → `services[i]`                        |
| Месячный доход (для Salary Guess)             | `src/data/roles.ts` → `monthlyIncome`                      |
| Вопросы и варианты ответа                     | `src/data/questions.ts`                                    |
| Иконки ролей                                  | `src/data/roles.ts` → `iconName` (имя иконки lucide)       |
| Акцент и градиент карточки роли               | `src/data/roles.ts` → `accent` и `gradient`                |

В билингвальных полях `{en: "...", ru: "..."}` нужно заполнить **оба** ключа. Если один из них пустой, отображается английский (EN — фолбэк).

### Как добавить новую профессию

1. В `src/data/roles.ts` добавить новый объект в массив `ROLES` со стабильным `id` (kebab-case).
2. Заполнить все билингвальные поля (`en` + `ru`).
3. В `src/data/questions.ts` проставить веса для нового `id` в тех опциях, куда он вписывается.
4. UI подхватит автоматически — ничего в компонентах менять не нужно.

### Скоринг теста

`src/lib/scoring.ts` → функция `scoreAnswers(answers, questions, roles)`:
- Для каждого ответа суммирует веса каждой роли из выбранной опции.
- Сортирует роли по убыванию суммы (тай-брейкер — `id` по алфавиту, детерминированно).
- Процент считается относительно лучшей суммы (топ всегда 100%).
- Возвращает массив `RoleResult[]` со всеми ролями, отсортированный — страница Result берёт топ-3.

## Кастомизация дизайна

Токены — в `tailwind.config.js`. Цвета (`ink-*`, `bone-*`, `amber`, `neon`, `iris`, `lime`), шрифты (`font-display`, `font-sans`, `font-mono`), дисплейные размеры (`text-display-xl/lg/md`), тени (`shadow-glow`, `shadow-glow-cyan`, `shadow-inner-soft`), анимации (`animate-fade-up`, `animate-pulse-soft`, `animate-shimmer`, `animate-ticker`).

Утилиты из `src/index.css`: `.glass`, `.glass-strong`, `.lumen`, `.grain::before`, `.text-balance`.

## Доступность

- Все интерактивные элементы навигируются с клавиатуры.
- Варианты ответа в тесте — настоящий `role="radiogroup"` со стрелочной навигацией и пробелом/Enter для выбора.
- Контрастные цвета поверх тёмного фона.
- `prefers-reduced-motion` учитывается (NeuralBackground, переходы).

## Известные особенности

- Состояние теста живёт в памяти — обновил страницу, пройдёшь заново. Это осознанное решение (подарок, а не SaaS).
- Bundle ~1.4 MB (с lucide и recharts). Для локального подарка это нормально.
- Кнопка «Поделиться» использует Web Share API, если доступен, иначе — копирует в буфер обмена.

## Деплой на Vercel

1. Импортировать репозиторий на [vercel.com/new](https://vercel.com/new) — Vercel автоматически определит **Vite** как framework preset.
2. Проверить параметры сборки: **Build Command** — `npm run build`, **Output Directory** — `dist`, **Install Command** — `npm install`.
3. Нажать **Deploy**. SPA-роуты покрыты `vercel.json` (все пути резолвятся на `index.html`, чтобы `react-router` работал при прямом заходе).
